📋 Step-by-Step Google Cloud Configuration Guide (A to Z)
Step 1: Go to Google Console & Create Project 1. Open the Google Cloud Console Credentials Page.
2. If you don't have a project, click the project dropdown in the top bar (next to 'Google Cloud'), click New Project, name it "Gemini Archive", and click Create. Select the project once created.
Step 2: Setup Consent Screen (One-time only) 1. Click OAuth consent screen in the left sidebar.
2. Choose External as User Type and click Create.
3. Enter App name (e.g., "Gemini Archive"), choose your email for User support email, and type your email at the bottom for Developer contact information.
4. Click Save and Continue on the App Information page.
5. Click Save and Continue on the Scopes page.
6. On the Test Users / Audience page, click + ADD USERS, type your Gmail address (e.g., your email), click Save, and then click Save and Continue.
7. Click Back to Dashboard.
Step 3: Create OAuth Client ID Credentials 1. Click Credentials on the left-hand menu.
2. Click the + Create Credentials button at the top, and select OAuth client ID.
3. Set Application type to Web application.
4. Under Authorized redirect URIs at the bottom, click + ADD URI and paste this exact link: loading... 5. Click Create at the bottom.
Step 4: Save & Sync 1. Copy the Client ID shown in the success popup (looks like a long text ending with .apps.googleusercontent.com).
2. Paste it into the input box above.
3. Click Sync with Google Drive below. A Google account pop-up window will open. Select your Google account and authorize the application!
